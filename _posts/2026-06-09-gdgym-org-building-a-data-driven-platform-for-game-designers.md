# GDGym.org – Building a Data-Driven Platform for Game Designers

## **My Role**

Product Owner, Product Analyst, Data Analyst, Content Creator, Python Developer.

## **Project Snapshot**

* Developed and maintain an independent platform for game designers.  
* Published 7 detailed deconstructions of popular and classic games.  
* Structured and documented approximately 100 game design elements.  
* Configured 8 custom GA4 events and tags to track user behavior.  
* Implemented a custom engagement funnel to measure content exploration and user contributions.  
* Configured analytics for both Google Search traffic and on-site search behavior.  
* Use YouTube content as a user acquisition channel and analyze its impact on engagement metrics.  
* Built automated content preparation workflows using Python, JSON, and Generative AI tools.

## **Project Overview**

As a Data Analyst, I believe it is important to understand not only how to analyze data but also how the systems generating that data work. This includes websites, client-server architecture, analytics platforms, content management workflows, and digital product development.

To deepen my understanding of these areas and stay up to date with modern technologies and methodologies, I developed [GDGym.org](http://GDGym.org) during my postgraduate studies in Game Design at IADE University in Lisbon. Today, I continue to maintain and expand the platform as an independent side project.

[GDGym.org](http://GDGym.org) is a platform designed for game designers and game design students. The website combines characteristics of a media platform and a user-generated content platform. It helps users analyze existing games, understand game design patterns, and generate ideas for new game concepts.

Users can:

* Explore [game deconstructions](https://www.gdgym.org/games/)  to understand how successful games are structured.  
* Discover alternative implementations of existing game elements.  
* Analyze how specific mechanics and design patterns are used across different games. Try [Core Loop](https://www.gdgym.org/elements/331/view) for example.  
* Contribute their own variations of game elements.  
* Evaluate and like ideas submitted by other users.  
* Participate in game design challenges (currently in development).

## **Product Challenge**

The main product challenge is increasing user engagement.

I view the platform as a knowledge-sharing ecosystem where users should move beyond passive content consumption and actively participate in content creation and evaluation.

My current product goals are:

* Increase average engagement time.  
* Increase time spent on high-value content pages.  
* Encourage deeper exploration of game-related content.  
* Increase the number of user-generated contributions.  
* Increase interactions with community-created content.  
* Improve conversion from readers to contributors.

To support these goals, I follow a Data-Driven Decision Making approach and use analytics to identify opportunities for product improvement.

## **Content Development**

Building a meaningful knowledge base was one of the first priorities for the project.

To establish an initial content foundation, I:

* Created and edited 7 detailed deconstructions of popular and classic games.  
* Structured and documented approximately 100 game design elements.  
* Organized relationships between games and design elements to support exploration and idea generation.  
* Developed workflows that allow new content to be added efficiently and consistently.

The content structure was designed to encourage users to move between related concepts and discover new game design ideas.

## **Analytics Infrastructure**

To support data-driven product development, I implemented a custom analytics framework using:

* Google Analytics 4 (GA4)  
* Google Tag Manager (GTM)  
* Google Search Console (GSC)

These tools allow me to monitor:

* User acquisition.  
* User engagement.  
* Content performance.  
* Search visibility.  
* Navigation behavior.  
* Conversion through key engagement funnels.

I configured 8 custom GA4 events and tags to track critical user actions across the platform.

Key tracked events include:

* Viewing a game deconstruction.  
* Viewing a game element page.  
* Searching for content.  
* Submitting a new variation of a game element.  
* Liking a community-submitted variation.  
* Additional engagement-related actions throughout the platform.

## **Funnel Analysis**

One of the primary objectives is understanding how users progress from consuming content to actively contributing.

To measure this journey, I designed a custom engagement funnel:

Game Deconstruction View → Game Element View → User Contribution into the Creation of the New Version

The funnel helps identify:

* Where users lose interest.  
* Which content successfully drives deeper exploration.  
* Which pages encourage contributions.  
* Opportunities to improve conversion between engagement stages.

This framework allows me to evaluate whether product changes increase the likelihood that users become active contributors rather than passive readers.

## **Search Analytics**

Search behavior provides valuable insight into user intent.

To better understand what users are looking for, I implemented analytics for both external and internal search activity.

### **Google Search Analytics**

Using Google Search Console, I track:

* Search queries that drive traffic to the platform.  
* Pages with the highest search visibility.  
* Opportunities for new content creation.  
* Search trends related to game design topics.

This information helps prioritize future content based on actual user demand.

### **On-Site Search Analytics**

I also track search queries entered directly on the website.

This allows me to:

* Identify information users cannot easily find.  
* Detect gaps in the existing content database.  
* Prioritize new game deconstructions and game elements.  
* Better understand user interests and intentions.

The combination of external and internal search analytics helps guide content strategy and product development decisions.

## **User Acquisition Experiments**

To attract new users, I create educational content related to game design on [YouTube](https://www.youtube.com/@MikaelPintassilgo).

Beyond generating traffic, my objective is to understand how different content types influence user behavior after arriving on the website.

Future analytics improvements include tracking:

* Which videos generate the highest-quality traffic.  
* Which traffic sources produce the most engaged users.  
* Which acquisition channels lead to user contributions.  
* How engagement differs depending on the entry point into the platform.

This will allow acquisition performance to be evaluated using behavioral metrics rather than traffic volume alone.

## **Automation and Data Workflows**

A significant portion of the platform's content creation process is automated.

Using Python and JSON-based workflows, I developed tools that transform game-related information into structured data suitable for publication on the website.

This approach reduces manual work, improves consistency, and enables faster content expansion.

Additionally, during my Master's studies at Nova IMS in Lisbon, I completed coursework focused on Generative AI. I apply this knowledge to accelerate content preparation and enrichment using large language models such as ChatGPT and Google Gemini.

The combination of automation and AI-assisted workflows allows the platform to scale content production while maintaining structure and quality.
